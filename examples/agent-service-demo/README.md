# 데모: OpenClaw — 잘 만들어진 AI Agent 서비스

> LO: `LO-Agent 설계 패턴과 운영` | 예상 시간: 2분

## 시연 목표

"잘 만들어진 AI Agent는 이런 겁니다 — IDE 없이 메시지 하나로 복잡한 작업 수행"

## 사전 체크리스트

```bash
# 1. 인터넷 연결 확인
curl -s -o /dev/null -w "%{http_code}" https://www.google.com
# → 200 이어야 함

# 2. OpenClaw 사이트 접속 확인
# 브라우저에서 OpenClaw 공식 사이트 접속 테스트
# 접속 불가 시: 사전 녹화 영상 또는 스크린샷 준비
```

## 시연 순서

### Step 1: Agent vs 챗봇 차이 설명 (30초)

| 구분 | 챗봇 (ChatGPT) | Agent (OpenClaw) |
|------|---------------|-----------------|
| 입력 | "검색해줘" | "이 주제를 조사해서 보고서 만들어줘" |
| 행동 | 답변만 | 검색 → 분석 → 문서 작성 → 이메일 전송 |
| 도구 | 없음 | 웹 브라우징, 파일, 이메일, 캘린더 |
| 자율성 | 묻고 답 | 스스로 판단하고 실행 |

### Step 2: OpenClaw 시연 (1분)

**브라우저에서 OpenClaw 데모 또는 녹화 영상 보여주기:**

1. WhatsApp/Slack으로 메시지 전송: "다음 주 월요일 팀 회의를 잡아줘. 참석자는 김팀장, 이대리, 박사원. 회의 안건은 Q1 리뷰"
2. Agent가 자율적으로:
   - 참석자 일정 확인
   - 가능한 시간대 탐색
   - 회의실 예약
   - 초대 메일 발송
   - 안건 문서 초안 작성

- "사람이 중간에 개입하지 않아도, Agent가 스스로 판단하고 실행합니다"
- "이게 방금 CrewAI에서 본 패턴의 실제 제품화 버전입니다"

### Step 3: 비즈니스 연결 (30초)

- "GitHub 스타 60K+ — 2달 만에 9K에서 60K"
- "이런 Agent 서비스가 산업 전반으로 확산되고 있습니다"
- "교육과정에서 이런 Agent를 설계하고 구축하는 방법을 배웁니다"

## 핵심 멘트

- "챗봇은 답만 합니다. Agent는 실행합니다"
- "Agent Washing 주의 — 단순 챗봇에 Agent 라벨을 붙이는 건 Agent가 아닙니다"
- "교육과정에서 진짜 Agent를 만듭니다"
